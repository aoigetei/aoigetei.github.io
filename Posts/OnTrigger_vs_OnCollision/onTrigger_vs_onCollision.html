OnTiggerEnter vs OnCollisionEnter

December 27
Detecting collisions and object overlap in Unity3D is widely handled using Colliders. Colliders can be used to apply a repelling force to colliding objects, or as a trigger to notify when objects overlap/intersect. Hence, we can process and respond to collision events using OnCollisionEnter or OnTriggerEnter. With that being said, the right time to use these functions might be confusing for some folks.

When onTriggerEnter is false, unity physiscs will prevent objects from entering each other. The OnCollisionEnter method is only called when none of the objects involved in the collision has isTrigger set to true.
